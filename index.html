Inuitcss for noobs

Inuitcss for noobs just does the following:

  1. Installs all the Inuitcss modules from their offical Github repo using Bower.
  2. Provides a default Sass file loading all the Inuitcss modules and activating several variables which are off by default.
  3. Compiles this Sass file into CSS with Grunt.
  4. Provides a preview HTML template using the compiled CSS.

Go on Github and follow the instructions if you would like to try.

Inuitcss is a CSS framework made by Harry Roberts. There is a getting started section on the official project github, but it doesn't give a clear overview of what the framework can do. I made this small project to learn more about it. The CSS file compiled here is not intended to be re-used like this. The authors made it clear that the philosophy of Inuitcss is to be "off" by default and each modules has to be included depending on the need of a specific project.

Settings

Global variables, site-wide settings, config switches, etc.

settings-defaults

Github: inuitcss/settings.defaults

The inuitcss defaults module contains a few variables and settings that are required for using any of the rest of the framework.

Install using Bower: 

$ bower install --save inuit-defaults

Variables:

  • $inuit-base-font-size: 16px !default;
  • $inuit-base-line-height: 24px !default;
  • $inuit-base-text-color: #333 !default;
  • $inuit-base-background-color: #fff !default;

settings-responsive

Github: inuitcss/settings.responsive

Define initial breakpoint aliases and conditions.

Install using Bower:

$ bower install --save inuit-responsive-settings

Tools

Site-wide mixins and functions.

tools-functions

Github: inuitcss/tools.functions

The inuitcss functions module contains a few framework functions that are required for using any of the rest of inuitcss.

Install using Bower:

$ bower install --save inuit-functions

tools-mixins

Github: inuitcss/tools.mixins

The inuitcss mixins module contains a few framework mixins that are required for using any of the rest of inuitcss.

Install using Bower:

$ bower install --save inuit-mixins

tools-responsive

Github: inuitcss/tools.responsive

Currently, this inuitcss module just sets up the media query mixin.

Install using Bower:

$ bower install --save inuit-responsive-tools

Generic

Low-specificity, far-reaching rulesets (e.g. resets).

generic-box-sizing

Github: inuitcss/generic.box-sizing

The inuitcss box-sizing module causes all elements to use the more useful border-box box model.

Install using Bower:

$ bower install --save inuit-box-sizing

generic-normalize

Github: inuitcss/generic.normalize

The normalize.css module is an exact clone of normalize.css at version v3.0.1. normalize.css makes browsers render all elements more consistently and in line with modern standards. It is developed and maintained by Nicolas Gallagher.

Install using Bower:

$ bower install --save inuit-normalize

generic-reset

Github: inuitcss/generic.reset

The inuitcss reset module is a considered approach to resetting elements. It selectively removes margins and paddings from certain elements, and provides some sensible defaults for some others.

Install using Bower:

$ bower install --save inuit-reset

generic.shared

Github: inuitcss/generic.shared

The inuitcss shared module contains several high-level rulesets which apply a consistent, shared declaration (typically margins) across a number of elements.

Install using Bower:

$ bower install --save inuit-shared

Unclassed HTML elements (e.g. a {}, blockquote {}, address {}).

base.headings

Github: inuitcss/base.headings

The inuitcss `headings` module defines font sizes for base-level heading elements from `h1` through to `h6`.

Objects

Objects, abstractions, and design patterns (e.g. .media {}).

objects.layout

The inuitcss layout system uses `box-sizing: border-box;` and `display: inline-block;` to create an extremely powerful, flexible alternative to the traditional grid system.
Combine the layout items with the widths found in `trumps.widths`.

Place any image- and text-like content side-by-side, as per: stubbornella.org/content/2010/06/25/the-media-object-saves-hundreds-of-lines-of-code

This is a <div> with class="media__body".

The flag object is a design pattern similar to the media object, however it utilises `display: table[-cell];` to give us control over the vertical alignments of the text and image. csswizardry.com/2013/05/the-flag-object
